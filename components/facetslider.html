Coveo component FacetSlider

The FacetSlider component allows to create a facet that renders a slider widget to filter on a range of numerical values rather than the classic multi-select facet with a label and a count for each values.
Note that this component does NOT inherit from a standard Facet, and thus does not offer all the same options. Also note that many options for the slider component cannot be set as an HTML attribute on the component, and must be configured in JavaScript.

A generic example on how to initialize a complex slider

// Using a JSON object inside the init call. Note that the JSON object follows the options described in this page.
Coveo.init(document.querySelector('#search'), {
   FacetSlider: {
     field: "@size",
     start: 1000,
     end: 5000,
     rangeSlider: true,
     graph: {
       steps: 10
     }
   }
})

// OR using the jquery extension

$('#search').coveo('init', {
   FacetSlider: {
     field: "@size",
     start: 1000,
     end: 5000,
     rangeSlider: true,
     graph: {
       steps: 10
     }
   }
})

// Same configuration, but using attribute directly on the element instead
<div class='CoveoFacetSlider' data-field='@syssize' data-start='1000' data-end='5000' data-range-slider='true' data-graph-steps='10'></div>

dateField

dateField: boolean
  • Defined in src/ui/FacetSlider/FacetSlider.ts :94

Specifies whether the field for which you are requesting a range is a date field.
This allows the facet to correctly build the outgoing group by request, as well as render it correctly.

Markup configuration example(s) :
data-date-field='true'
data-date-field='false'

dateFormat

dateFormat: string
  • Defined in src/ui/FacetSlider/FacetSlider.ts :114

Specifies the format used to display values if they are dates.
Default value is MMM dd, yyyy.

Markup configuration example(s) :
data-date-format='foo'

displayAsPercent

displayAsPercent: any
  • Defined in src/ui/FacetSlider/FacetSlider.ts :178

Specifies the percentage caption that you want to display the field values.
Available options are :

  • enable : (data-display-as-percent-enable) boolean : Specifies whether the caption should be displayed as a percentage. Default is false
    • .
  • separator : (data-display-as-percent-separator) string : Specifies the character(s) to use as a separator in the caption. Default is -.

displayAsValue: any
  • Defined in src/ui/FacetSlider/FacetSlider.ts :163

dropdownHeaderLabel

dropdownHeaderLabel: string
  • Defined in src/ui/FacetSlider/FacetSlider.ts :280

Specifies the label of the button that allows to show the facets when in responsive mode. If it is specified more than once, the first occurence of the option will be used. The default value is "Filters".

Markup configuration example(s) :
data-dropdown-header-label='foo'

enableResponsiveMode

enableResponsiveMode: boolean
  • Defined in src/ui/FacetSlider/FacetSlider.ts :267

Specifies if the responsive mode should be enabled on the facets. Responsive mode will make the facet dissapear and instead be availaible using a dropdown button. Responsive facets are enabled when the width of the element the search interface is bound to reaches 800 pixels. This value can be modified using Facet.options.responsiveBreakpoint. The default value is true.

Markup configuration example(s) :
data-enable-responsive-mode='true'
data-enable-responsive-mode='false'

end

end: string
  • Defined in src/ui/FacetSlider/FacetSlider.ts :132

Specifies the ending boundary of the slider.
Dates values are rounded on the year when the field used is a date type.
Optional: Takes the highest value available in the index by default.

Markup configuration example(s) :
data-end='foo'

excludeOuterBounds

excludeOuterBounds: boolean
  • Defined in src/ui/FacetSlider/FacetSlider.ts :137

Specifies if you want to exclude the outer bounds of your slider in the generated query, when they are not active.
Default value is false.

Markup configuration example(s) :
data-exclude-outer-bounds='true'
data-exclude-outer-bounds='false'

field

field: IFieldOption
  • Defined in src/ui/FacetSlider/FacetSlider.ts :100

Specifies the index field whose values will be use in the facet.
This require the given field to be configured correctly in the index as a facet field.
This is a required option and cannot be omitted, otherwise the facet component will not work.

Markup configuration example(s) :
data-field='@foo'

getSteps

getSteps: function
  • Defined in src/ui/FacetSlider/FacetSlider.ts :230

Specifies a function that will generate the steps for the slider. The function receives the slider boundaries and must return an array of number (the steps).

Coveo.init(document.querySelector('#search'), {
   FacetSlider: {
     field: "@size",
     getSteps: function(start, end) {
       return [0,2,4,6,8,10];
     }
   }
})

// OR using the jQuery extension

$('#search').coveo('init', {
   FacetSlider: {
       field: "@size",
       getSteps: function(start, end) {
           return [0,2,4,6,8,10];
       }
   }
})

    id

    id: string
    • Defined in src/ui/FacetSlider/FacetSlider.ts :107

    Specifies a unique identifier for a facet. This identifier will be used to save the facet state in the URL hash, for example.
    Optional, since the default will be the FacetSlider.options.field option.
    If you have two facets with the same field on the same page, you should specify an ID for at least one of those two facets.
    That ID needs to be unique on the page.

    Markup configuration example(s) :
    data-id='foo'

    queryOverride

    queryOverride: string
    • Defined in src/ui/FacetSlider/FacetSlider.ts :120

    Specifies the query to filter automatic minimum and maximum range of the slider.
    This is especially useful for date range, where the index may contain values which are not set, and thus the automatic range will return values from the year 1400 (min date from the boost C++ library).
    Can be used to do something like queryOverride : @date>2000/01/01 or some arbitrary date which will filter out unwanted values.

    Markup configuration example(s) :
    data-query-override='foo'

    rangeSlider

    rangeSlider: boolean
    • Defined in src/ui/FacetSlider/FacetSlider.ts :153

    Specifies whether you want a slider with two buttons, or only one.

data-range-slider='false'

    responsiveBreakpoint

    responsiveBreakpoint: number
    • Defined in src/ui/FacetSlider/FacetSlider.ts :274

    Specifies the width of the search interface, in pixels, at which the facets will go into responsive mode. The responsive mode will be triggered when the width is equal or below this value. The search interface corresponds to the element with the class CoveoSearchInterface. The default value is 800.

    Markup configuration example(s) :

    Optional: By default the slider will allow all values.
