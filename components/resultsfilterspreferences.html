Coveo component ResultsFiltersPreferences

This component allows end-users to create their own customized filters to apply to their search.
It is a feature meant for advanced user that understand the Coveo Query syntax.
These preferences are saved in the localStorage of each user.
This component is normally accessible visually using the Settings menu.
It is usually located, in the DOM, inside the PreferencesPanel component.

Component Options

filters

filters: any
  • Defined in src/ui/ResultsFiltersPreferences/ResultsFiltersPreferences.ts :94

Specifies default filters that all users will see.

End-users will not be able to modify or delete them, as they are not customized for each user, but instead created by the creator/developer of the search page.

Filters should follow this definition :

  filters: { [caption: string]: { expression: string; tab?: string[]; }};

eg :

Coveo.init(root, {
  ResultsFiltersPreferences : {
    filters : {
      'Only google drive stuff' : {
        tab: ['Tab1', 'Tab2'],
        expression : '@connectortype == 'GoogleDriveCrawler''
      },
      'Another filter' : {
        expression : 'another expression''
      }
    }
  }
})

This is optional, and by default this option is left 'empty'.

includeInBreadcrumb

includeInBreadcrumb: boolean
  • Defined in src/ui/ResultsFiltersPreferences/ResultsFiltersPreferences.ts :56

Specifies whether to include the active filter(s) in the breadcrumb.
Default is true.

Markup configuration example(s) :
data-include-in-breadcrumb='true'
data-include-in-breadcrumb='false'

showAdvancedFilters

showAdvancedFilters: boolean
  • Defined in src/ui/ResultsFiltersPreferences/ResultsFiltersPreferences.ts :62

Specifies whether to show or hide the Create button to allow end-users to create their own filters.
If set to false, only pre-populated filter created on initialization using js code will be available to the end user. Default is true.

Markup configuration example(s) :
data-show-advanced-filters='true'
data-show-advanced-filters='false'
