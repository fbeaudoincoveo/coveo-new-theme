Coveo component ResultAttachments

This component is used to render attachments in a result set, for example when displaying emails.
It is intended to be used inside a result template when there is an active Folding component inside the page.

Component Options

maximumAttachmentLevel

maximumAttachmentLevel: number
  • Defined in src/ui/ResultAttachments/ResultAttachments.ts :53

Specifies the maximum nesting depth at which the component should stop rendering sub-attachments.
The default value is 5.

Markup configuration example(s) :
data-maximum-attachment-level='10'

resultTemplate

resultTemplate: Template
  • Defined in src/ui/ResultAttachments/ResultAttachments.ts :41

Specifies the template to use to render each of the attachments for a top result.
By default, it will use the template specified in a child element with a <script> tag.
This can be specified directly as an attribute to the element, for example :

<div class="CoveoResultFolding" data-result-template-id="Foo"></div>

which will use a previously registered template ID (see TemplateCache)

subResultTemplate

subResultTemplate: Template
  • Defined in src/ui/ResultAttachments/ResultAttachments.ts :48

Specifies the template to use to render sub-attachments, which are attachments within other attachments, for example multiple files embedded in a .zip attachment.
Sub-attachments can also contain other sub-attachments.
The template can be specified the same way as resultTemplate
