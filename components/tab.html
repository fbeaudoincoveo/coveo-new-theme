Coveo component Tab

This component is a bar allowing users to select a search interface.

The component attaches itself to an div element and is in charge of adding an advanced expression to the query and thus, modify the outgoing query in order to refine the results in relation to the selected tab.

It also allows to hide and show different parts of the UI. In order to do so, each component of the UI can specify whether or not it wishes to be included or excluded from a specific tab.

Including and excluding other HTML components

You can hide/show specific HTML components, based on the currently selected search tab by adding the following attributes:

  • <div data-tab="foobar"> -> Include this element only in the tab with the ID 'foobar'.

  • <div data-tab-not="foobar"> -> DO NOT include this element in the tab ID 'foobar'.

  • <div data-tab="foobar,somethingelse"> -> Include this element only in the tab with the ID 'foobar' or 'somethingelse'.

Setting a new endpoint for a tab

A tab can use a custom endpoint when performing a query. First, you need to make sure that the endpoint exists in the array of Coveo.SearchEndpoint.endpoints.

Coveo.SearchEndpoint.endpoints["specialEndpoint"] = new Coveo.SearchEndpoint({
    restUri : 'https://somewhere.com/rest/search'
})

[...]

<div class='CoveoTab' data-endpoint='specialEndpoint'></div>

constant

constant: boolean
  • Defined in src/ui/Tab/Tab.ts :107

Specifies whether the filter expression should be included in the constant part of the query.
The constant part of the query is specially optimized by the index to execute faster, but you must be careful not to include dynamic query expressions otherwise the cache would lose its efficiency.
By default, this option is set to true.

Markup configuration example(s) :
data-constant='true'
data-constant='false'

enableResponsiveMode

enableResponsiveMode: boolean
  • Defined in src/ui/Tab/Tab.ts :134

Specifies if the responsive mode should be enabled for the tabs. Responsive mode will make the overflowing tabs dissapear and instead be availaible using a dropdown button. Responsive tabs are enabled when tabs overflow or when the width of the search interface becomes too small.

Disabling reponsive mode for one tab will disable it for all tabs. Therefore, this options only needs to be set on one tab to be effective. The default value is true.

Markup configuration example(s) :
data-enable-responsive-mode='true'
data-enable-responsive-mode='false'

maximumAge

maximumAge: number
  • Defined in src/ui/Tab/Tab.ts :124

Specifies the maximum age in milliseconds that cached query results can have in order to be used (instead of performing a new query on the index).
The cache is located in the Coveo Search API (which resides between the index and search interface).
If cached results are available but are older than the specified age, a new query will be performed on the index.
By default, this is left undefined and the Coveo Search API will decide the cache duration.

Markup configuration example(s) :
data-maximum-age='10'

pipeline

pipeline: string
  • Defined in src/ui/Tab/Tab.ts :117

Specifies the name of the query pipeline to use for the queries, in the Coveo platform ( Query Pipeline in the Cloud administration).
If not specified, the default value is null, in which case pipeline selection conditions defined in a Coveo Cloud organization apply.

Markup configuration example(s) :
data-pipeline='foo'

sort

sort: string
  • Defined in src/ui/Tab/Tab.ts :101

Specifies the default sort when this tab is selected. A Sort component configured with the same specified parameter needs to be in the interface in order for this this option to function properly.
e.g.: relevancy / date descending
Optional, by default the normal Sort component behavior will operate.

Markup configuration example(s) :
data-sort='foo'
